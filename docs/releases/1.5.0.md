# RightKeyboard 1.5.0

Fecha: 2026-07-09

## Resumen

RightKeyboard 1.5.0 es la versión estable inicial de la línea 1.5. Consolida la identificación persistente de teclados, la configuración visual en WinUI 3, el instalador por usuario y las correcciones de Raw Input acumuladas durante las alfas y betas.

## Cambios principales

- Configuración WinUI 3 bajo demanda, separada del proceso residente que escucha Raw Input.
- Selector de distribución renovado, con alias de dispositivo, agrupación visual por idioma y opción para ignorar periféricos ambiguos.
- Identidad persistente basada en `ContainerId`, `InstanceId` o ruta HID, más huella conservadora para recuperar preferencias cuando Windows entrega datos suficientes.
- Persistencia en `%LOCALAPPDATA%\RightKeyboard\preferences.json`, con migración desde la configuración heredada.
- Menú nativo de bandeja limitado a **Configuración** y **Salir**.
- Acción **Limpiar preferencias** integrada en la ventana de Configuración.
- Instalador Inno Setup por usuario, sin elevación, con acceso directo de menú Inicio e inicio automático por defecto en instalación nueva.
- Diagnóstico detallado retirado del build normal; el código queda disponible para compilaciones de desarrollo mediante símbolo explícito.
- Correcciones de foco, activación y estados visuales del selector, Configuración y confirmaciones modales respecto de las betas previas.

## Pendientes aceptados para mantenimiento

Esta lista describe el estado en la fecha de publicación y se conserva como registro. Varias entradas ya se abordaron en el trabajo posterior, todavía sin publicar; se anota al lado de cada una.

- Robustecer el diagnóstico de desarrollo y documentar una variante reproducible para pruebas de host. **Sigue pendiente.**
- Mejorar la detección preventiva de periféricos HID que se presentan como teclado, especialmente presentadores USB, mouse avanzados y dispositivos virtuales. **Implementado después de 1.5.0** como exclusión por firma HID parcial; falta validarlo con el presentador físico.
- Evaluar una agrupación manual de identidades para el mismo teclado cuando Windows lo distingue por puerto USB. **Implementado después de 1.5.0**; falta validarlo con dos teclados físicos y cambio de puerto.
- Migrar exportación/importación y administración de inicio con Windows a la Configuración WinUI. **Implementado después de 1.5.0**; la portabilidad entre equipos sigue sin certificar.
- Revisar microtextos de la interfaz que aún sean demasiado descriptivos o técnicos. **Sigue pendiente.**

El estado vigente y el orden de ejecución están en [plan-1.6.0.md](../plan-1.6.0.md).

## Notas para pruebas posteriores

Los dispositivos HID ambiguos pueden seguir abriendo el selector si Windows los reporta como teclados. En esos casos, la acción **Ignorar este dispositivo** continúa siendo el mecanismo seguro de respaldo.

La primera detección de un dispositivo nuevo sigue requiriendo una tecla utilizable; una pulsación aislada de modificadores como `Ctrl`, `Shift`, `Alt` o `Windows` no inicia el selector.
